# FSM Anomaly Detector


Анализатор переходов конечных автоматов (Finite State Machine) для выявления аномальных паттернов поведения в логах телекоммуникационных систем.

> *Проект выполнен в рамках хакатона по C++ «Выход на связь»*  
> *18 апреля 2026 · Протей БЦ «Телеком» · Санкт-Петербург, Б. Сампсониевский пр., 60*


## Функционал

- **Потоковая обработка** лог-файлов (без загрузки всего файла в память)
- **Парсинг** FSM-логов с выделением состояний и входящих сообщений
- **Обнаружение зависших FSM** — автоматов, не достигших конечного состояния
- **Поддержка файла конечных состояний** для валидации терминальных состояний
- **Генерация CSV-отчета** с аномалиями в требуемом формате
- **Индикатор прогресса** обработки больших файлов
- **Эффективное использование памяти** — хранение только активных FSM

## Требования

- C++17 компилятор (g++ 7+, clang 6+)
- CMake 3.15+
- Linux / macOS / Windows (WSL)

## Архитектура

- **Parser** — Парсинг строк лога с использованием регулярных выражений
- **Analyzer** — Ядро анализатора — отслеживание состояний FSM, обнаружение аномалий
- **Reporter** — Генерация CSV-отчета о зависших FSM
- **StateMachine** — Модель конечного автомата (имя, ID, состояние, последнее сообщение)

## Формат входных данных

### log_file

Строки лога содержат два типа событий:

#### 1. Смена состояния (State Change)

```bash
2026-01-12 09:07:02.346 2513826 PrimFSM.cpp(72) FSM: Sg.SIP.UA id: 2621; < St: 9 WAITING_PRACK
```

**Поля:**
- `2026-01-12 09:07:02.346` — timestamp
- `Sg.SIP.UA` — имя FSM
- `2621` — ID экземпляра FSM
- `WAITING_PRACK` — новое состояние

#### 2. Входящее сообщение (Incoming Message)

```bash
2026-01-12 09:07:02.345 2513826 PrimFSM.cpp(37) FSM: Sg.SIP.UA id: 2621; > St: 5 INCOMING_DIALOG_PROCEEDING Pr: 38418:4 SIP_UA_ALERTING_REQ ()
```

**Поля:**
- `2026-01-12 09:07:02.345` — timestamp
- `Sg.SIP.UA` — имя FSM
- `2621` — ID экземпляра FSM
- `INCOMING_DIALOG_PROCEEDING` — текущее состояние
- `SIP_UA_ALERTING_REQ` — входящее сообщение

### end_states_file

Содержит перечень терминальных состояний для каждого типа FSM:
```bash
Sg.SIP.UA : NUL
```

**Поля:**
- `Sg.SIP.UA` — Имя конечного автомата
- `NUL` — Состояние, считающееся нормальным завершением

## Формат выходных данных

### output_file

```bash
# Пример строки из отчета
2026-01-12 09:07:02.346 Sg.SIP.UA 2621 WAITING_PRACK SIP_UA_ALERTING_REQ 00:00:00.000
```

| Поле | Имя | Формат | Описание |
|------|-----|--------|----------|
| `2026-01-12 09:07:02.346` | timestamp | `YYYY-MM-DD HH:MM:SS.mmm` | Время последнего изменения состояния |
| `Sg.SIP.UA` | logic name | строка | Имя FSM |
| `2621` | logic id | число | Идентификатор экземпляра FSM |
| `WAITING_PRACK` | state | строка | Текущее состояние FSM |
| `SIP_UA_ALERTING_REQ` | last message | строка | Последнее входящее сообщение |
| `00:00:00.000` | duration | `HH:MM:SS.mmm` | Время нахождения в данном состоянии |

## Алгоритм обнаружения аномалий

1. **Инициализация** — загрузка файла конечных состояний
2. **Потоковое чтение** — построчная обработка лог-файла
3. **Парсинг** — определение типа события (смена состояния / входящее сообщение)
4. **Обновление состояния** — сохранение актуального состояния FSM
5. **Финальная проверка** — после обработки всего файла, FSM, не достигшие конечного состояния, считаются зависшими
6. **Генерация отчёта** — создание CSV-файла с аномалиями

## ✨ Ключевые особенности

| Особенность | Описание |
|:------------|:---------|
| **Потоковая обработка** | Файл читается построчно, не загружается в память целиком |
| **O(1) по памяти** | Хранятся только активные FSM, не вся история |
| **Регулярные выражения** | Быстрый и гибкий парсинг строк лога |
| **Индикатор прогресса** | Визуальная обратная связь при обработке больших файлов |
| **Точность времени** | Поддержка миллисекунд в timestamp и длительности |
| **Минималистичный дизайн** | Только нужные поля, без оверхэда |
